### Researcher batch engagement

R36 `_researcher_factory` captured the bare `_spawn` reference at
factory time and handed it to `make_researcher_batch(original_fn)`.
Path A v1/v2 wrapped `spawn_wrappers["spawn_researcher"]` (singular)
but the batch tool `spawn_researchers([...])` invoked the captured
bare `_spawn` directly ‚Äî bypassing the cancel-null gate entirely.
Result: every `spawn_researchers([...])` call on a parent thread
with a leaked `cancel_event` aborted on iter-1 entry.

`c2acafc` adds `_cybergym_cancel_null(bare_spawn)` and applies it
inside both `_researcher_factory` and `_analyzer_factory` BEFORE
handing `original_fn` to the batch builders. R37 numbers:

| task | researcher.jsonl entries | ABORTED |
|---|---|---|
| arvo:1065 | (no researcher rollout ‚Äî model didn't use batch) | n/a |
| arvo:16634 | (no researcher rollout) | n/a |
| arvo:38393 | 16 | 0 |
| arvo:48736 | 48 | 0 |

R36 baseline: arvo:1065 had 9 researcher.jsonl entries, **6 ABORTED**
(67% abort rate). R37: every researcher invocation that ran (16 +
48 = 64 entries) completed normally.

## Mechanism recap

Path A v1 (`e339f3f`) ‚Äî wrap singular `spawn_<name>` factories with
a gate that nulls `_cancel_event` on the **wrapped chain return**.
No-op for verifier/critic/fixer because their spawn_hooks
`make_<X>_spawn_wrapper` returns plain closures that don't forward
`__setattr__`.

Path A v2 (`5b337af`) ‚Äî thread bare `original_fn` through factories
so the null happens on the **bare _spawn** whose `_cancel_event`
attribute is the one actually read at `agent.py:148`. Fixed
verifier/critic/fixer. Singular `spawn_analyzer` and
`spawn_researcher` also fixed (but model uses plural batch tools).

Path A v2 extension (`c2acafc`) ‚Äî `_cybergym_cancel_null` shim
applied at `_researcher_factory` and `_analyzer_factory` BEFORE
passing `original_fn` into `make_researcher_batch(...)` /
`make_analyzer_spawn_wrapper(...)`. Per-call null+restore around
`bare_spawn(*args, **kwargs)`. No-op when `KAI_BENCHMARK != cybergym`
so bountybench/evmbench keep normal cancel propagation.

Tests: 6 in `tests/test_cybergym_cancel_decouple.py` ‚Äî singular
gates wrap all 5 sub-agents, exception path restores, batch shim
nulls + restores, batch shim is no-op outside cybergym, unsettable
attribute handled.

## What this does NOT fix

* Model PoC quality. 3/4 tasks engaged the harness extensively
  (‚â•17 verifier iters) but only `arvo:1065`'s softmagic MSAN bug
  produced a real crash. `arvo:16634` ran 46 verifier iters with
  100+ harness POSTs (visible in ngrok window) and no crash.
  `arvo:48736` reached `soft_verified` on a UTF-8 OOB hypothesis
  but the verifier couldn't confirm the PoC bytes triggered a
  crash. Future work: tighter PoC-byte-generation prompt or
  fuzzing-mutator helper.
* Long-tail post-pipeline stalls. `arvo:38393` heartbeated for
  179min then died with critic only at 2 entries ‚Äî possibly stuck
  in `chain_assembler` or `fixer` waiting on an LLM call. Not a
  cancel-leak issue.
