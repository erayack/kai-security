## Operating Rules

1. **ALWAYS use `list_files()` with optional depth parameter**: Default depth is 2. Use higher values for deeper exploration.

2. **Git submodules - PROACTIVE APPROACH**:
   - **ALWAYS check for SSH URLs FIRST**: Before running git_submodule_update(), use convert_ssh_to_https_in_gitmodules() to prevent SSH failures
   - **If submodules fail with SSH errors**: Run convert_ssh_to_https_in_gitmodules() and retry git_submodule_update()
   - **Don't give up after SSH failures**: Convert SSH to HTTPS and retry - most submodules will then succeed
   - Empty third_party/ directories mean you need to run git_submodule_update()

3. **Rust workspace dependency failures - WORKAROUND REQUIRED**:
   - **If cargo build fails with "failed to load source for dependency"**: This means a dependency (often from a failed submodule) is missing
   - **Solution**: Use create_minimal_cargo_package() to create placeholder packages
   - **Example**: If error says "failed to read bft/manytrace/agent/Cargo.toml", create minimal package:
     ```python
     result = create_minimal_cargo_package("bft/manytrace/agent", "agent")
     result = create_minimal_cargo_package("bft/manytrace/tracing-manytrace", "tracing-manytrace")
     ```
   - Then retry cargo_build() - the workspace will now compile even without the missing dependency
   
4. **Build order matters**: Usually: convert SSH → submodules → create placeholders → dependencies → configure → build

5. **Test incrementally**: After each major step, verify it worked before proceeding

6. **Read documentation**: Check README.md and build scripts before attempting setup

7. **Handle errors gracefully**: If a step partially fails, analyze what succeeded and continue with workarounds:
   - SSH failures → convert_ssh_to_https_in_gitmodules() and retry
   - Missing Cargo dependencies → create_minimal_cargo_package() and retry
   - Build tool failures → try alternative approaches or skip that component

8. **ALWAYS respond with <think> tags**: Even after errors, ALWAYS start your response with <think>...</think> before providing <python> or <done>

9. **NEVER give up prematurely**: Try ALL available workarounds before using <done>. The next agents depend on you!
