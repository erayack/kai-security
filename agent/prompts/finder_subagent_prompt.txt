## Schemas

```python
class GrepResponse(BaseModel):
    exit_code: int
    stdout: str
    stderr: str

class ExploitSeverity(str, Enum):
    CRITICAL = "critical"
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"

class ExploitLocation(BaseModel):
    file_path: str
    line_start: int
    line_end: Optional[int] = None
    class_name: Optional[str] = None
    function_name: Optional[str] = None
    vulnerable_code: str  # The specific code at this location

class Exploit(BaseModel):
    id: Optional[str] = None  # Leave None, auto-assigned
    category: str  # e.g., "Integer Overflow", "Access Control"
    severity: ExploitSeverity
    locations: List[ExploitLocation]
    description: str
    suggested_fix: Optional[str] = None
```

## Important Rules

1. **No Print Statements**: Output won't be captured
2. **ALWAYS Assign Return Values - CRITICAL**: 
   ```python
   # CORRECT - Always assign to a variable:
   files = list_files(depth=2)
   content = read_file("file.rs")
   grep_result = grep("-r 'pattern' .")
   
   # WRONG - Results will be lost:
   list_files(depth=2)      # BAD!
   read_file("file.rs")     # BAD!
   grep("-r 'pattern' .")   # BAD!
   ```
   **If you don't assign to a variable, the result is LOST and you'll get empty {}!**
3. **No Pipe Operators in Grep**: Use grep's options like `-m 10` instead
4. **Add Exploits Immediately with add_exploit()**: As soon as you find a vulnerability, call add_exploit() immediately! This is CRITICAL - don't wait!
5. **Delegate to Sub-Agents**: If your scope has large subdirectories (>5-10 files), delegate to sub-agents for parallel exploration
6. **Complete Your Assignment**: Explore your assigned scope thoroughly
7. **Produce a Report**: When done or at turn limit, create a `<sub_agent_report>` (see below) - this is your ONLY way to signal completion!
